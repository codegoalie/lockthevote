Authentication

Authentication

Authenticate and get session tokens

POST /token

The token endpoint requires an email address and password for a user and returns a token response which contains an authentication token to be used when accessing other API endpoints as well as a refresh token which can be used to get a new authentication token without requiring the user to enter their credentials again. This provides continual access to the API without a long lived token nor the clients to store and access the user's credentials.

Email address of the user.

Elections

Elections

Lists current elections the user can participate

GET /elections

The elections endpoint returns elections in which the user can, or could, participate. Withput providing a state value, only open elections will be returned. Closed or upcoming elections can be fetched using the 'state' parameter.

state

State(s) for which returned elections will match

string[]
in

Voting

Voting

Casts a vote in a race in an election

POST /vote

Given a race and a selection, casts a vote on bahalf of the authenticated user in the given race.

race_id

Race identifier provided in the /elections endpoint.

type
integer
in
query
selection

Candidate identifier(s) also provided in the /elections endpoint. Either a single candidate identifier, or a comma-separated list of identifiers, depending on the race type. Race needs a single identifier. PickTwoRace needs at most two identifiers. RankedRace needs at most the number of candidates in the race in the order of preference.

type
string
in
query
authentication-token

Authentication token from the /token endpoint.

type
string (BEARER {authentication-token})
in
header
201 Created
Vote

Vote cast

400 Bad Request
Error

Vote not cast due to error in the vote request submitted. Error specifics will be included in the response, but examples include improper selection format or attempting to vote on a closed race.

Response Example (201 Created) 
{
  "race_id": "integer",
  "selection": "string"
}
Response Example (400 Bad Request) 
{
  "code": "integer(int32)",
  "messages": "array"
}

Models

Candidate: object

id: integer

Unique identifer of the candidate to be used when voting

name: string

Name of the candidate

Election: object

name: string

Title of the election

starts_at: string

ISO-8601 timestamp when the election opens

ends_at: string

ISO-8601 timestamps when the election closes

races: object[]

Error: object

code: integer (int32)
messages: string[]

Race: object

id: integer

Unique identifier of the race to be used when voting

office: string

Title of the office at stake in the race

description: string

Description of the office at stake in the race

type: string

Specific type of race, Race (pick one), PickTwoRace, or RankedRace

candidates: object[]

Token: object

authentication_token: string

Authentication token to be used when accessing other API endpoints.

expires: string

This is the ISO-8601 formatted expiration of the authentication token.

refresh_token: string

Refresh token to be used to get a new authentication token after the authentication token expires.

refresh_expires: string

ISO-8601 formatted expiration time of the refresh token.

Vote: object

race_id: integer

Identifier of the race for which the vote was cast

selection: string

Identifier(s) for the candidate to which the vote was cast
